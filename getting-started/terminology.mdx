This glossary covers the core terminology used in Factory Flow. Understanding these concepts will help you navigate the platform and configure it for your factory.

## Core entities

### Organization

A multi-tenant workspace that contains all your factory data. Each organization has:
- Isolated data that's not visible to other organizations
- Its own set of users with role-based permissions
- Separate API tokens and billing
- Independent settings and configurations

Users can belong to multiple organizations and switch between them.

### Facility

A physical location such as a factory, plant, building, or site. Facilities help you:
- Organize machines by location
- Filter reports and analytics by site
- Group resources for multi-site operations

Example: "Main Plant", "Building A", "Austin Facility"

### Machine

A production asset that generates data. Machines are the primary entities you'll track in Factory Flow. Each machine has:
- A unique identifier (UUID)
- A type (e.g., CNC Mill, Injection Molder, Assembly Station)
- An assigned facility
- Optional line and shift assignments
- A configured data source for ingestion
- Analytics thresholds for cycle time monitoring

### Production line

A sequence of machines that work together to produce parts. Lines help you:
- Visualize material flow between machines
- Track line-level metrics and throughput
- Organize machines by process

Example: "Assembly Line 1", "Paint Line", "Final Packaging"

### Layout

A visual representation of your factory floor. Layouts include:
- Uploaded blueprint or floor plan images
- Machine positions with X/Y coordinates
- Dimensions and scale
- Machine connections and flow visualization

Use layouts to quickly see your entire floor and navigate to specific machines.

### Shift

A defined work period for scheduling and capacity planning. Shifts include:
- Start and end times
- Day-of-week configuration
- Machine assignments

Examples: "Day Shift (6 AM - 2 PM)", "Night Shift (10 PM - 6 AM)", "Weekend Crew"

## Data and analytics

### Part count

The number of parts produced by a machine. Factory Flow tracks three types:
- **Total parts** - All parts produced
- **Good parts** - Parts that passed quality checks
- **Bad parts** - Parts that were rejected or scrapped

Part counts are submitted via the ingestion API and stored in time-series format.

### Cycle time

The time required to complete one production cycle. This includes:
- **Actual cycle time** - Measured time between parts
- **Expected cycle time** - Target time configured for the machine
- **Cycle time variance** - Difference between actual and expected

### Downtime

Periods when a machine is not producing. Downtime is calculated from gaps in production data and categorized for Pareto analysis.

### OEE (Overall Equipment Effectiveness)

A manufacturing standard for measuring productivity. OEE combines:
- **Availability** - Actual production time vs. planned time
- **Performance** - Actual output vs. theoretical output
- **Quality** - Good parts vs. total parts

OEE = Availability × Performance × Quality

### Hypertable

A time-series database table optimized for high-volume production data. Factory Flow uses TimescaleDB hypertables to store:
- Live part counts
- Cycle time measurements
- Downtime events

This enables fast queries across large time ranges.

## Production concepts

### SKU (Stock Keeping Unit)

A product identifier used to track what's being manufactured. SKUs are associated with:
- Bills of materials
- Work orders
- Quality specifications

### BOM (Bill of Materials)

The list of components and quantities required to build a product. BOMs define:
- Raw materials needed
- Sub-assemblies required
- Quantities per unit

### Work order

A production order specifying what to make and how many. Work orders include:
- SKU to produce
- Quantity required
- Due date
- Assigned production line

### WIP (Work in Progress)

Parts that are currently being manufactured but not yet complete. WIP tracking helps you:
- Monitor production flow
- Identify bottlenecks
- Calculate lead times

### Hold

A paused state applied to work orders or inventory due to:
- Quality issues requiring investigation
- Missing materials or components
- Equipment problems

## Users and access

### User

An individual who has access to Factory Flow. Users authenticate with email or OAuth and can belong to one or more organizations.

### Role

A permission level assigned to a user within an organization:

| Role | Permissions |
|------|-------------|
| **Viewer** | Read-only access. Can view dashboards, analytics, and reports. |
| **Editor** | Can create and modify machines, lines, shifts, facilities, and layouts. |
| **Admin** | Full access including member management, API tokens, billing, and settings. |

### Invitation

An email sent to invite someone to join an organization. Invitations include:
- The target email address
- The assigned role
- An expiration time

### Join request

A request from a user to join an organization they found by searching. Admins can approve or deny these requests.

## API and integration

### API token

A credential used to authenticate API requests. Tokens are:
- Scoped to a specific organization
- Assigned a permission level (None, Read, Read/Write)
- Displayed only once when created

Store tokens securely; they cannot be retrieved after creation.

### Data source

The method by which a machine receives production data. The primary data source is the Factory Flow API, which accepts JSON payloads at a per-machine endpoint.

### Ingestion endpoint

The API endpoint for submitting production data:

```
POST /api/machines/{machine_uuid}/parts
```

Accepts part counts and records them with timestamps in the time-series database.

### Webhook

An HTTP callback triggered by events in Factory Flow. Webhooks can notify external systems when:
- Machine status changes
- Production thresholds are exceeded
- Data ingestion events occur

## Billing

### Subscription

The active billing plan for an organization. Subscriptions are managed through Stripe and control:
- Access to features
- Machine limits
- Data retention periods

### Trial period

An initial period for new organizations to evaluate Factory Flow with full access before committing to a paid plan.

### Grace period

A limited access period after subscription cancellation, allowing time to export data or resubscribe.

### Billing portal

A Stripe-hosted interface where admins can:
- Update payment methods
- View invoices
- Manage subscription changes
- Download receipts

## User interface

### Overview

The main dashboard showing organization-wide status:
- Machine counts and connection status
- Recently updated machines
- Machines needing attention
- Recent layouts

### Sidebar

The left navigation panel providing access to all major sections:
- Overview
- Machines
- Lines
- Shifts
- Facilities
- Layouts
- Analytics
- Organization Settings

### Toast notification

A brief message that appears to confirm actions or alert you to issues. Toasts appear in the corner of the screen and dismiss automatically.
