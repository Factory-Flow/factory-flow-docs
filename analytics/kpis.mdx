Factory Flow calculates and displays key performance indicators (KPIs) that help you measure and improve manufacturing efficiency. This guide explains each metric and how to use them effectively.

## OEE (Overall Equipment Effectiveness)

OEE is the gold standard for measuring manufacturing productivity. It combines three factors into a single percentage.

### Formula

```
OEE = Availability × Performance × Quality
```

### Components

| Component | Formula | Measures |
|-----------|---------|----------|
| **Availability** | Run Time ÷ Planned Production Time | How often the machine runs |
| **Performance** | (Ideal Cycle Time × Total Count) ÷ Run Time | How fast it runs when running |
| **Quality** | Good Count ÷ Total Count | How many parts are good |

### OEE benchmarks

| Score | Assessment |
|-------|------------|
| <40% | Significant improvement needed |
| 40-60% | Typical for untracked equipment |
| 60-85% | Good, room for improvement |
| 85-100% | World class |

### Understanding OEE losses

OEE identifies six major losses:

**Availability losses:**
- Breakdowns (unplanned stops)
- Setup and adjustments

**Performance losses:**
- Small stops and idling
- Reduced speed

**Quality losses:**
- Startup rejects
- Production rejects

## Cycle time

Cycle time measures how long it takes to produce one unit.

### Types of cycle time

| Type | Definition |
|------|------------|
| **Actual cycle time** | Measured time between parts |
| **Expected cycle time** | Target time configured for the machine |
| **Ideal cycle time** | Theoretical minimum time |

### Calculating cycle time

Factory Flow calculates cycle time from the timestamps of consecutive part counts:

```
Cycle Time = Time of Current Part - Time of Previous Part
```

### Cycle time thresholds

Configure thresholds to visualize performance:

| Zone | Meaning | Color |
|------|---------|-------|
| On target | Within expected time | Green |
| Warning | Slightly over target | Yellow |
| Critical | Significantly over target | Red |

### Using cycle time data

- **Identify variation** - Inconsistent cycle times indicate process issues
- **Find bottlenecks** - Longest cycle time limits line output
- **Track improvement** - Monitor trends over time
- **Set realistic targets** - Base expectations on actual data

## Throughput

Throughput measures the rate of production output.

### Throughput metrics

| Metric | Definition |
|--------|------------|
| **Parts per hour** | Output rate |
| **Parts per shift** | Shift-level production |
| **Daily output** | Total production per day |

### Calculating throughput

```
Throughput = Total Parts ÷ Time Period
```

### Theoretical vs. actual

- **Theoretical throughput** - Based on expected cycle time
- **Actual throughput** - Based on measured output
- **Throughput efficiency** - Actual ÷ Theoretical

## Downtime

Downtime tracks when machines are not producing.

### Types of downtime

| Type | Description |
|------|-------------|
| **Planned downtime** | Scheduled maintenance, breaks |
| **Unplanned downtime** | Breakdowns, stoppages |
| **Idle time** | Machine available but not running |

### Downtime metrics

| Metric | Formula |
|--------|---------|
| **Total downtime** | Sum of all non-production time |
| **MTBF** | Mean time between failures |
| **MTTR** | Mean time to repair |

### Downtime Pareto

The downtime Pareto chart ranks causes by duration:
- Identifies the biggest time losses
- Focuses improvement efforts
- Tracks reduction over time

## Quality metrics

Quality metrics track production defects and yield.

### Key quality metrics

| Metric | Formula |
|--------|---------|
| **Good parts** | Parts passing inspection |
| **Bad parts** | Rejected/scrapped parts |
| **Quality rate** | Good Parts ÷ Total Parts × 100% |
| **First pass yield** | Parts correct first time |
| **Scrap rate** | Bad Parts ÷ Total Parts × 100% |

### Quality tracking in Factory Flow

When sending data via the API, report:
- `part_count_total` - All parts produced
- `part_count_good` - Parts that passed
- `part_count_bad` - Parts that failed

Factory Flow calculates quality rate automatically.

## Availability

Availability measures the percentage of planned time that production occurs.

### Formula

```
Availability = Run Time ÷ Planned Production Time × 100%
```

### Components

| Component | Definition |
|-----------|------------|
| **Planned production time** | Total shift time minus planned stops |
| **Run time** | Actual production time |
| **Downtime** | Planned production time minus run time |

### Improving availability

- Reduce unplanned breakdowns
- Minimize changeover time
- Optimize maintenance scheduling
- Track and address recurring issues

## Performance

Performance measures how fast equipment runs compared to its capability.

### Formula

```
Performance = (Ideal Cycle Time × Total Count) ÷ Run Time × 100%
```

### Performance losses

- **Minor stoppages** - Brief stops not recorded as downtime
- **Slow cycles** - Running below designed speed
- **Operator delays** - Human-related slowdowns

### Improving performance

- Identify and address slow cycles
- Reduce minor stoppages
- Standardize operating procedures
- Provide operator training

## How Factory Flow calculates metrics

### Data sources

Metrics are calculated from ingested data:

```json
{
  "part_count_total": 100,
  "part_count_good": 98,
  "part_count_bad": 2
}
```

### Time-series storage

Data is stored in TimescaleDB hypertables:
- `machine_live_parts` - Raw part counts
- `machine_analytics_cycle_times` - Calculated cycle times
- `machine_analytics_downtimes` - Detected downtime events

### Aggregation

Metrics can be aggregated by:
- Time period (hour, shift, day, week)
- Machine
- Production line
- Facility

## Using metrics effectively

### Setting targets

1. **Baseline first** - Measure current performance
2. **Set realistic goals** - Incremental improvement
3. **Track progress** - Monitor trends over time
4. **Celebrate wins** - Acknowledge improvements

### Focus areas

| Goal | Key metric |
|------|------------|
| Reduce stops | Availability, Downtime |
| Increase speed | Performance, Cycle time |
| Improve quality | Quality rate, Scrap rate |
| Overall improvement | OEE |

### Avoiding common mistakes

- **Don't game metrics** - Focus on real improvement
- **Consider context** - Compare similar conditions
- **Look at trends** - Single data points can mislead
- **Address root causes** - Don't just treat symptoms
