Production lines represent the logical grouping of machines that work together to manufacture products. Use lines to organize your factory floor, visualize material flow, and track aggregate performance metrics.

## Understanding production lines

A production line in Factory Flow represents:

- A sequence of machines that parts flow through
- A logical grouping for capacity planning
- An aggregation point for line-level analytics
- A way to organize related equipment

Lines help you:

- **Track throughput** across multiple machines
- **Identify bottlenecks** in the production process
- **Calculate line-level OEE** and efficiency
- **Plan capacity** based on line capabilities
- **Visualize material flow** with the flow editor

## Creating a production line

To create a new line:

1. Navigate to **Lines** in the sidebar
2. Click **Create Line**
3. Fill in the line details:

| Field | Description |
|-------|-------------|
| **Name** | A unique name for the line (e.g., "Assembly Line 1", "Paint Line") |
| **Part Number** | The product/part this line produces |
| **Description** | Optional notes about the line's purpose |

4. Click **Create**

## Line configuration

### Basic settings

After creating a line, configure its properties:

- **Name** - Update the display name
- **Part Number** - The SKU or part identifier produced
- **Description** - Notes about the line, process, or specifications

### Machine assignments

Add machines to your production line:

1. Open the line detail page
2. Navigate to the **Machines** or **Flow** section
3. Add machines from your organization to the line
4. Arrange them in production sequence

## The flow editor

Factory Flow includes a visual flow editor for designing line configurations.

### Using the flow editor

The flow editor provides a canvas where you can:

1. **Add nodes** - Start points, machines, and end points
2. **Create connections** - Draw edges between nodes to show material flow
3. **Arrange layout** - Drag nodes to organize the visual representation
4. **Configure nodes** - Set properties for each element

### Node types

| Node Type | Purpose |
|-----------|---------|
| **Start** | Entry point for materials/parts |
| **Machine** | Production equipment in the line |
| **End** | Exit point for finished parts |

### Creating connections

To connect nodes:

1. Click on the output of a source node
2. Drag to the input of a destination node
3. Release to create the connection

Connections show the direction of material flow through the line.

### Editing the flow

To modify an existing flow:

- **Move nodes** - Drag nodes to new positions
- **Delete connections** - Click an edge and press Delete
- **Remove nodes** - Select and delete unwanted nodes
- **Add machines** - Insert new machines into the sequence

## Managing lines

### Viewing lines

The Lines page displays all production lines:

- Line name and part number
- Number of machines
- Last updated timestamp

### Editing lines

1. Click on a line to open its detail page
2. Modify settings, machines, or flow configuration
3. Click **Save** to apply changes

### Deleting lines

1. Navigate to the line's detail page
2. Click **Delete**
3. Confirm the deletion

<Warning>
  Deleting a line does not delete the machines assigned to it. Machines are disassociated but remain in your organization.
</Warning>

## Line analytics

Production lines aggregate analytics from their member machines.

### Line-level metrics

View aggregate performance data:

- **Total throughput** - Parts produced across all machines
- **Line OEE** - Overall equipment effectiveness for the line
- **Bottleneck identification** - See which machine limits output
- **Downtime impact** - How machine downtime affects line performance

### Accessing line analytics

1. Navigate to **Analytics**
2. Switch to **Line Analytics** view
3. Select a line from the dropdown
4. View aggregated performance data

### Comparing machines

Within a line, compare:

- Individual machine cycle times
- Machine-specific OEE
- Relative performance between stations
- Utilization rates

## Capacity planning

Use lines for capacity planning:

### Calculating capacity

Based on line configuration:

- **Bottleneck rate** - The slowest machine determines maximum output
- **Shift capacity** - Output possible during defined shifts
- **Planned downtime** - Account for maintenance windows

### Demand matching

Compare capacity to demand:

- Forecast requirements by part number
- Identify capacity gaps
- Plan additional shifts or equipment

## Best practices

### Line organization

- **One product per line** - Simplifies tracking and analytics
- **Logical sequences** - Mirror the physical production flow
- **Clear naming** - Include product or process identifier

### Machine assignment

- **Complete coverage** - Include all machines in the process
- **Correct sequence** - Ensure flow reflects reality
- **Single assignment** - Each machine typically belongs to one line

### Flow editor tips

- **Start simple** - Begin with main machines, add detail later
- **Use consistent spacing** - Improves readability
- **Label clearly** - Help team members understand the layout
- **Update regularly** - Keep the flow current with changes

### Naming conventions

| Good | Why |
|------|-----|
| "Assembly Line 1 - Widget A" | Product-specific |
| "Paint Line - Building A" | Location context |
| "Pack Line - Priority" | Function and priority |

## Common configurations

### Serial line

Parts flow through machines in sequence:

```
Start → Machine A → Machine B → Machine C → End
```

### Parallel stations

Multiple machines perform the same operation:

```
           → Machine A →
Start →                    → End
           → Machine B →
```

### Branching lines

Different paths for different products:

```
                    → Machine B → End (Product 1)
Start → Machine A →
                    → Machine C → End (Product 2)
```
