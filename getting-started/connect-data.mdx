---
title: "Connect machine data"
description: "Generate a token, configure a data source, and send a test payload."
---

This step verifies that Factory Flow can receive production data from your equipment.

## Create an API token

<Steps>
  <Step title="Open API tokens">
    Navigate to **Organization Settings → API Tokens**.
  </Step>
  <Step title="Create the token">
    Click **Create Token**, name it, and choose **Read & Write** permissions.
  </Step>
  <Step title="Store it securely">
    Tokens are only shown once. Copy it into your PLC, gateway, or integration service.
  </Step>
</Steps>

## Configure the machine data source

<Steps>
  <Step title="Select Factory Flow API">
    On the machine detail page, open the **Data Source** tab and choose **Factory Flow API**.
  </Step>
  <Step title="Copy the machine UUID">
    You'll use the UUID in your ingestion endpoint.
  </Step>
</Steps>

## Send a test data point

```bash
curl -X POST https://app.factoryflow.com/api/machines/{machine_uuid}/parts \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "part_count_total": 1,
    "part_count_good": 1,
    "part_count_bad": 0
  }'
```

Replace `{machine_uuid}` and `YOUR_API_TOKEN` with your values.

<Note>
  For full payload options, validation rules, and more examples, see the [Data ingestion](/analytics/ingestion) guide.
</Note>
