Follow these steps to go from a fresh account to seeing live analytics on your first machine.

<Steps>
  <Step title="Create your account">
    Head to [app.factoryflow.app/register](https://app.factoryflow.app/register) and sign up with your work email.

    You'll receive a one-time code by email to verify your address. Enter it on the verification screen, then set your password to complete registration.

    <Tip>
      If your team is already on Factory Flow, ask your Organization Admin to send you an invitation instead. Accepting an invitation automatically connects your account to your team's workspace.
    </Tip>

    <Frame>
      <img className="block dark:hidden" src="/images/getting-started/register.png" alt="Create your account registration form" />
      <img className="hidden dark:block" src="/images/getting-started/register-dark.png" onError={(e) => { e.currentTarget.src = "/images/getting-started/register.png"; }} alt="Create your account registration form" />
    </Frame>
  </Step>

  <Step title="Set up your organization">
    After signing in for the first time you'll be prompted to either **create a new organization** or **join an existing one**.

    - **Create** — Enter a name for your workspace (usually your company or plant name). This becomes the shared environment for your entire team.
    - **Join** — Search for your company's existing organization. Your request will be sent to an admin for approval and you will be notified once a decision is made.

    <Note>
      Everything in Factory Flow — machines, lines, analytics, and team members — lives inside an organization.
    </Note>
  </Step>

  <Step title="Create your first facility">
    Navigate to **Configure → Facilities** and click **New Facility**.

    Enter a facility name. Facilities represent physical locations — a plant, warehouse, or building. If you only have one site, create one facility for it.

    <Frame>
      <img className="block dark:hidden" src="/images/getting-started/quickstart-new-facility.png" alt="New Facility form" />
      <img className="hidden dark:block" src="/images/getting-started/quickstart-new-facility-dark.png" onError={(e) => { e.currentTarget.src = "/images/getting-started/quickstart-new-facility.png"; }} alt="New Facility form" />
    </Frame>
  </Step>
  

  <Step title="Create your first machine">
    Navigate to **Configure → Machines** and click **New Machine**.

    Fill in:
    - **Name** — A clear, recognizable label (e.g. "Press Line 3" or "CNC-04").
    - **Machine Type** — The category this machine belongs to (Lathe, Press, Robot, etc.).

    <Frame>
      <img className="block dark:hidden" src="/images/getting-started/create-new-machine.png" alt="New Machine form" />
      <img className="hidden dark:block" src="/images/getting-started/create-new-machine-dark.png" onError={(e) => { e.currentTarget.src = "/images/getting-started/create-new-machine.png"; }} alt="New Machine form" />
    </Frame>

    After saving, your machine will appear in the machines list. Navigate to the machine, click **Edit** and scroll down to the **Data Source** section and select the appropriate data souce for the machine.

    Options are:
    - **Factory Flow API** — Send real-time metrics using the Factory Flow ingestion endpoint. Requires an API token with read & write permissions.
      <Note>
        To use the Factory Flow API, you'll need an organization API token. Go to **Manage → Organization → API Tokens** and click **New Token**. Give it a name, set permissions to **Read & Write**, then copy it — it's only shown once. Use this token wherever you're sending data from (Node-RED, etc.). [See more →](/manage/organization#api-tokens)
      </Note>
      <Note>
        View the full API reference to see available endpoints, request formats, and example payloads. [View API reference →](/api/introduction)
      </Note>
    - **Factory Flow Gateway** — Receive real-time metrics from a Factory Flow Gateway device connected to this machine.
      <Note>
        To use a Factory Flow Gateway, it must first be registered in your organization. Go to **Manage → Organization → Gateways** and click **New Gateway**. You'll receive a unique identifier to configure on the gateway device. [See more →](/manage/organization#gateways)
      </Note>

    <Frame>
      <img className="block dark:hidden" src="/images/getting-started/machine-data-source.png" alt="Data source configuration" />
      <img className="hidden dark:block" src="/images/getting-started/machine-data-source-dark.png" onError={(e) => { e.currentTarget.src = "/images/getting-started/machine-data-source.png"; }} alt="Data source configuration" />
    </Frame>
  </Step>

  <Step title="Create a production line">
    Navigate to **Configure → Lines** and click **New Line**.

    Give the line a name and assign it to your facility. Once saved, open the line and use the **Part Flow** tab to drag your machines into the sequence they run — left to right represents the order parts flow through the line.

    <Frame>
      <img className="block dark:hidden" src="/images/getting-started/quickstart-part-flow.png" alt="Part Flow diagram editor" />
      <img className="hidden dark:block" src="/images/getting-started/quickstart-part-flow-dark.png" onError={(e) => { e.currentTarget.src = "/images/getting-started/quickstart-part-flow.png"; }} alt="Part Flow diagram editor" />
    </Frame>
  </Step>

  <Step title="Set up a shift schedule">
    Navigate to **Configure → Shifts** and click **New Shift**.

    Choose a preset (8-hour, 12-hour, or 24-hour) or drag to draw custom shift windows on the weekly schedule grid. Assign this shift to your machines so Factory Flow knows when production is expected to be running.

    <Tip>
      Shift schedules affect OEE calculations. Only time inside a defined shift counts toward Availability — downtime outside of shifts is not penalised.
    </Tip>

    <Frame>
      <img className="block dark:hidden" src="/images/configure/demo.factoryflow.io_shifts_6f0b0813-13a3-44dd-a0d3-11430d9d9be9.png" alt="Shift schedule weekly grid" />
      <img className="hidden dark:block" src="/images/configure/demo.factoryflow.io_shifts_6f0b0813-13a3-44dd-a0d3-11430d9d9be9-dark.png" onError={(e) => { e.currentTarget.src = "/images/configure/demo.factoryflow.io_shifts_6f0b0813-13a3-44dd-a0d3-11430d9d9be9.png"; }} alt="Shift schedule weekly grid" />
    </Frame>
  </Step>

  <Step title="View your analytics">
    Navigate to **Monitor → Analytics** and select your machine, line, or facility from the dropdown.

    You'll see live KPI tiles and trend charts as data starts flowing in. Use the time window selector in the top-right to zoom between 1 hour and 3 months of history.

    <Frame>
      <img className="block dark:hidden" src="/images/getting-started/quickstart-analytics.png" alt="Analytics dashboard with KPI tiles and charts" />
      <img className="hidden dark:block" src="/images/getting-started/quickstart-analytics-dark.png" onError={(e) => { e.currentTarget.src = "/images/getting-started/quickstart-analytics.png"; }} alt="Analytics dashboard with KPI tiles and charts" />
    </Frame>
  </Step>

  <Step title="Set up your first alert">
    Navigate to **Configure → Alert Rules** and click **New Alert Rule**.

    Choose what to monitor (a machine, line, or facility), set a threshold (e.g. "Cycle Time exceeds 45 seconds"), and assign an alert group to receive email notifications when the rule fires.
  </Step>
</Steps>
