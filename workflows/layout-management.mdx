This workflow guides you through creating and managing factory layouts in Factory Flow. Layouts provide a visual representation of your factory floor that helps teams navigate, monitor status, and understand production flow.

## When to use layouts

Create layouts when you need to:

- **Visualize your factory floor** - See machine positions at a glance
- **Monitor real-time status** - Display machine status on wall screens
- **Navigate quickly** - Click machines to access their data
- **Communicate changes** - Plan and share floor reorganizations
- **Support new team members** - Help them understand the factory

## Step 1: Plan your layout

Before creating a layout in Factory Flow, gather the necessary information:

### Preparation checklist

- [ ] Obtain a floor plan or blueprint (CAD export, architectural drawing, or sketch)
- [ ] List the machines to include
- [ ] Identify machine positions on the floor
- [ ] Decide on the level of detail (entire floor vs. specific area)

### Choosing the right scope

| Scope | Best for |
|-------|----------|
| **Entire facility** | Overview dashboards, visitor presentations |
| **Single floor/building** | Day-to-day operations, shift monitoring |
| **Production area** | Detailed line management, troubleshooting |

## Step 2: Create the layout

1. Navigate to **Layouts** in the sidebar
2. Click **Create Layout**
3. Configure the layout settings:

| Setting | Recommendation |
|---------|----------------|
| **Title** | Clear, descriptive name (e.g., "Building A - Level 1") |
| **Width** | Match your floor plan proportions |
| **Height** | Match your floor plan proportions |
| **Description** | Optional notes about purpose or scope |
| **Visibility** | Start with Private; make Public when ready |

4. Click **Create**

## Step 3: Upload your blueprint

Add a background image to provide context:

1. Click **Upload Blueprint**
2. Select your floor plan image (JPG or PNG)
3. Wait for the upload to complete
4. The image appears as the layout background

<Tip>
  **Image tips:**
  - Use high resolution (1920x1080 or higher) for clarity when zooming
  - Remove unnecessary details to keep focus on production areas
  - Ensure consistent orientation (e.g., north up)
</Tip>

## Step 4: Position machines

Add machines to the layout and position them accurately:

### Adding machines

1. Click **Add Machine** in the layout editor
2. Select a machine from your organization
3. The machine appears on the canvas
4. Repeat for all machines to include

### Positioning machines

1. Click and drag each machine to its physical location
2. Align machines with the blueprint background
3. Resize machines if needed (drag corner handles)
4. Adjust Z-Index for overlapping machines

### Accuracy tips

- **Use gridlines** if your blueprint has them
- **Position consistently** - all machines same relative size
- **Match reality** - positions should reflect the actual floor
- **Update when machines move** - keep layouts current

## Step 5: Configure machine display

Customize how machines appear on the layout:

### Display options

Each machine can show:
- Machine name
- Current status (connected/disconnected)
- Real-time cycle time
- Color-coded performance indicators

### Setting cycle time display

1. Select a machine on the layout
2. Enable cycle time display in properties
3. The machine shows live cycle time values
4. Colors reflect configured thresholds

## Step 6: Review and publish

Before sharing your layout:

### Review checklist

- [ ] All machines are positioned correctly
- [ ] Machine names are visible and readable
- [ ] Status indicators are working
- [ ] Blueprint provides useful context
- [ ] Layout title and description are accurate

### Publishing the layout

1. Review the layout in edit mode
2. Click **Save** to apply changes
3. If ready for team use, change visibility to **Public**
4. Share the layout URL with team members

## Managing layouts over time

### Regular updates

Keep layouts current by:
- Adding new machines when installed
- Removing machines when decommissioned
- Repositioning machines when moved
- Updating blueprints after floor changes

### Version management

Consider creating new layouts for major changes:

| Scenario | Approach |
|----------|----------|
| Minor machine move | Update existing layout |
| Floor reorganization | Create new layout, archive old |
| Proposed changes | Create draft layout for planning |
| Historical reference | Keep old layouts for comparison |

### Layout naming for versions

Use naming conventions to track versions:
- "Building A - Current" - Active layout
- "Building A - Q1 2024 Proposed" - Planning version
- "Building A - Pre-Expansion" - Historical reference

## Using layouts for monitoring

### Wall displays

Create layouts for real-time floor monitoring:

1. Design a layout specifically for wall screens
2. Focus on status visibility over detail
3. Use larger machine icons for visibility at distance
4. Enable auto-refresh in the browser

### Shift handoffs

Use layouts during shift changes:

1. Open the layout at the start of shift
2. Review machine status colors
3. Click machines showing issues
4. Document observations in handoff notes

### Quick navigation

Use layouts for rapid access:

1. Click any machine on the layout
2. View quick stats in the popup
3. Click through to the machine detail page
4. Access analytics and configuration

## Best practices

### Design principles

- **Keep it simple** - Include only relevant machines
- **Maintain accuracy** - Positions should match reality
- **Update regularly** - Stale layouts lose trust
- **Use good blueprints** - Quality backgrounds help orientation

### Organization

- **One layout per area** - Don't overcrowd layouts
- **Consistent naming** - Follow a naming convention
- **Clear descriptions** - Help others understand the purpose
- **Archive, don't delete** - Keep historical layouts when relevant

### Team adoption

- **Train users** - Show how to navigate and click through
- **Gather feedback** - Ask what would be helpful
- **Place on displays** - Make layouts visible to floor teams
- **Celebrate updates** - Acknowledge when layouts are refreshed

## Troubleshooting

### Common issues

| Issue | Solution |
|-------|----------|
| Blueprint too blurry | Upload higher resolution image |
| Machines not showing status | Verify data source is configured |
| Layout loads slowly | Reduce blueprint file size |
| Can't find a machine | Check that machine belongs to your organization |

### Getting help

If you encounter issues with layouts:
- Check that machines have data sources configured
- Verify your browser supports the layout features
- Contact [support@factoryflow.com](mailto:support@factoryflow.com)
